Hexo搭建个人博客

2018/11/26

老早之前就想搭建一个自己的博客，但出于各种原因都没能静下心来开搞，今天硬逼着自己弄完了，下面分享下搭建步骤。

一、搭建Hexo本地博客

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。安装Hexo之前需要安装node.js和git,具体的安装方法请自行百度谷歌，在此不再赘述。

使用npm安装Hexo

npm install -g hexo-cli

是不是很简单呢？安装 Hexo 完成后，执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件：

hexo init blog //blog 你的项目文件夹
cd blog //切换到站点根目录
npm install //安装所需文件

新建完成后，指定文件夹的目录如下：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
| ├── _drafts
| └── _posts
└── themes

每个文件的说明请自行查阅Hexo官方文档，在此不再赘述。所需文件安装后我们就可以使用以下命令在本地开启 Hexo 服务，在地址栏访问我们的本地博客了：

hexo generate //生成静态文件，可简写为 hexo g
hexo server //启动服务器。可简写为 hexo s
默认情况下，访问网址为： http://localhost:4000/

至此我们的Hexo本地博客就搭建完成，可通过 http://localhost:4000/ 访问。

二、将本地博客上传到github

新建 git 仓库

此步默认大家都注册好了 gitHub 账号，登陆自己的 gitHub 账号在主页点击绿色的 New repository 新建项目，注意，项目名必须为： 用户名.github.io
如下图：

我先前已经建立了这个仓库，所以才会报该仓库已存在的错误，大家可忽略。
填写完各项内容后点击 Create repository 按钮完成项目新建，到此我们就可以通过 用户名.github.io 访问到我们新建的项目了。

本地部署到GitHub

  • 首先我们要修改本地 Hexo 的配置文件 _config.yml ; 将文件中的 deploy 配置项修改为下图所示，注意空格，地址为你的地址：

  • 部署到GitHub。 真正部署前我们需要先装个插件：

    npm install hexo-deployer-git –save

  • 然后执行部署命令：

    hexo –deploy //文件生成后立即部署网站

静待几秒，部署完成后我们就可以通过 用户名.github.io 看到我们自己写的博客了。

三、发布自己的第一篇博文

我们使用 git 工具进入到本地项目的根目录，然后使用以下命令新建博客：

hexo new “博文名字” //简写 hexo n （新建的文件后缀为 .md）
该文件在 /blog/source/_posts 目录下

使用 IDE 工具编辑完成博文后，使用以下命令部署：

hexo g //生成静态页面
hexo d //发布

部署完就可以愉快的访问了。

四、绑定自己的域名

此步骤非必须，因为我们的博客已经可以通过 GitHub 提供的地址访问了，但总感觉逼格不高，鉴于域名比较便宜就在阿里云购买了一个。域名买好后需要实名认证，这些都搞完后我们就可以进行下面的操作了：

  • 域名解析
    记录类型： CNAME
    主机记录： WWW
    解析线路： 选默认就行
    记录值： 为你的域名
    TTL： 默认就行

  • 仓库设置

  1. 在 GitHub 上打开博客仓库设置
  2. 在 Custom domain下，填写自己的域名，点击 save
  3. 在站点目录的 source 文件夹下，创建并打开CNAME.txt，写入你的域名（如 www.xxx.com ),保存，然后将后缀改为CNAME
  4. 等待几分钟后即可通过自己的域名访问博客了
